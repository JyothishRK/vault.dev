OpenTelemetry:
-----

* OpenTelemetry (OTel) is a standard framework to collect telemetry data from applications:
    - Metrics
    - Logs
    - Traces
* Acts like a instrumentation layer within application to capture the data.
* With OpenTelemetry:

        Application
           ↓
        OpenTelemetry SDK
           ↓
        Metrics / Logs / Traces
           ↓
        Prometheus / Loki / Tempo
           ↓
        Grafana

* Single instrumentation layer compatible with all tools (no need of separate SDKs for each tool).
* By using OpenTelemetry, we extract Metrics, Logs & Traces (no need of separate setups).
* OpenTelemetry Collector acts a router to collect, batch and export the data to corresponding tool making the architecture clean.

        Apps
          ↓
        OTel Collector
          ├── Prometheus
          ├── Tempo
          ├── Loki
          └── Datadog

* After adding the collector the architecture becomes simple:

   App → OTel Collector → Everything else
